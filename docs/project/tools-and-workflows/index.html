Tools and Workflows

This document outlines recommended tools and workflows for the Light City project across different stages and aspects of development.

Project Management Tools​

Documentation​

  • Primary: Markdown files in Git repository
    • Simple, version-controlled, portable
    • Works offline, owns your data
    • Great for technical documentation
  • Alternative: Notion, Obsidian, or Craft
    • Better for visual organization
    • Rich media embedding
    • Database functionality

Task Management​

  • Simple: GitHub Issues or plain markdown TODO files
  • Structured: Trello, Asana, or Linear
  • Collaborative: Monday.com or ClickUp
  • Personal: Things, Todoist, or Notion

Recommendation for Stage 1: Keep it simple with markdown TODO lists and GitHub Issues (if using Git).

Version Control​

  • Git + GitHub/GitLab/Bitbucket
    • Essential for code and documentation
    • Enables collaboration
    • Provides backup and history

Creative Tools​

Visual Design & Concept Art​

2D Design:

  • Figma - UI/UX design, collaborative
  • Adobe Photoshop/Illustrator - Professional design
  • Affinity Designer/Photo - One-time purchase alternative
  • Procreate (iPad) - Digital illustration
  • Canva - Quick graphics and presentations

3D Modeling:

  • Blender - Free, powerful, open source (recommended)
  • SketchUp - Architectural modeling
  • Cinema 4D or Maya - Professional 3D
  • Rhino + Grasshopper - Parametric design

AI-Assisted:

  • Midjourney - Concept art generation
  • DALL-E - Image generation
  • Stable Diffusion - Open source image generation
  • RunwayML - Creative AI tools

Architectural Design​

  • SketchUp - Quick architectural modeling
  • Revit or ArchiCAD - Professional BIM
  • Rhino - Complex geometry
  • AutoCAD - Technical drawings
  • Blender - Free alternative for everything

Diagramming​

  • Miro or Mural - Collaborative boards
  • Lucidchart - Professional diagrams
  • draw.io - Free, simple diagrams
  • Figma - Design + diagrams
  • Notion or Obsidian - Text-based with diagrams

Writing & Documentation​

Writing Tools​

  • VS Code - For markdown with Git
  • iA Writer or Ulysses - Focused writing
  • Notion - Rich documents with databases
  • Obsidian - Linked knowledge base
  • Google Docs - Collaborative editing

Documentation Systems​

  • Static Site Generators:
    • Docusaurus - Documentation websites
    • GitBook - Beautiful documentation
    • MkDocs - Python-based docs
    • VitePress - Vue-powered docs
    • Hugo - Fast static sites

Development Tools (Stage 3+)​

Web Development​

Frontend:

  • React or Vue - Modern web apps
  • Next.js or Nuxt - Full-stack frameworks
  • Svelte - Lightweight alternative
  • Three.js or Babylon.js - 3D in browser
  • Tailwind CSS - Utility-first styling

Backend:

  • Node.js + Express - JavaScript backend
  • Python + FastAPI - Python backend
  • Supabase or Firebase - Backend-as-a-service
  • Netlify or Vercel - Easy deployment

Interactive Experiences​

  • Unity - Game engine, VR/AR
  • Unreal Engine - High-fidelity 3D
  • WebXR - Web-based VR/AR
  • A-Frame - HTML-based VR
  • p5.js - Creative coding

Communication & Community​

Community Platforms​

  • Discord - Real-time chat, voice, community
  • Slack - Professional team communication
  • Circle or Mighty Networks - Community platform
  • Forum - Discourse or custom forum
  • Social Media - Instagram, Twitter, etc.

Video & Meetings​

  • Zoom - Video meetings
  • Gather.town - Virtual spaces
  • Discord - Voice channels
  • Loom - Async video messages

Research & Knowledge Management​

Research Tools​

  • Notion - Database of research
  • Obsidian - Linked notes, local-first
  • Zotero - Academic research management
  • Raindrop.io or Pocket - Bookmark management
  • Google Scholar - Academic papers

Knowledge Organization​

  • Second Brain approach (Tiago Forte)
  • Zettelkasten method (slip-box)
  • PARA method (Projects, Areas, Resources, Archive)
  • Wiki-style linking (Obsidian, Notion)

Media Production​

Audio​

  • Ableton Live or Logic Pro - Music production
  • Audacity - Free audio editing
  • Suno or Udio - AI music generation

Video​

  • Final Cut Pro or Adobe Premiere - Professional editing
  • DaVinci Resolve - Free professional editor
  • iMovie - Simple video editing
  • ScreenFlow - Screen recording + editing

Recommended Workflow by Stage​

Stage 1: Foundation & Vision​

Core Tools:

  1. Markdown editor (VS Code, iA Writer, or Obsidian)
  2. Git for version control
  3. Simple task list (markdown file or GitHub Issues)
  4. Mood boarding (Pinterest, Are.na, or local folder)
  5. Basic sketching (Paper, Procreate, or Figma)

Workflow:

  1. Write core concepts in markdown
  2. Collect visual inspiration
  3. Create rough sketches and diagrams
  4. Organize in clear folder structure
  5. Version control everything

Stage 2: World-Building & Design​

Add:

  • 3D modeling (Blender or SketchUp)
  • Concept art tools (Photoshop, Procreate, or AI tools)
  • Diagramming (Miro, Lucidchart, or draw.io)

Workflow:

  1. Create detailed architectural plans
  2. Develop concept art and visualizations
  3. Build 3D models of key structures
  4. Design city layout with diagrams
  5. Document everything

Stage 3: Digital Presence​

Add:

  • Web development tools (Next.js, Tailwind, etc.)
  • Static site generator (Docusaurus, GitBook)
  • Analytics (Google Analytics, Plausible)
  • SEO tools (Ahrefs, SEMrush)

Workflow:

  1. Choose web framework
  2. Design website in Figma
  3. Develop and deploy website
  4. Set up social media
  5. Launch publicly

Stage 4: Community & Collaboration​

Add:

  • Discord or community platform
  • Collaborative tools (Miro, Notion)
  • Video meeting tools (Zoom)
  • Newsletter (Substack, ConvertKit)

Workflow:

  1. Set up Discord/community space
  2. Create onboarding materials
  3. Host regular gatherings
  4. Facilitate collaborations
  5. Document community wisdom

Stage 5+: Physical & Living System​

Add:

  • Project management (Asana, Monday)
  • Fundraising tools (GoFundMe, Patreon)
  • CRM (HubSpot, Notion)
  • Financial tools (QuickBooks, Wave)

Tool Selection Principles​

1. Start Simple​

  • Don't over-tool in early stages
  • Use what you know
  • Add complexity as needed

2. Own Your Data​

  • Prefer open formats (markdown, plain text)
  • Use Git for version control
  • Export regularly from proprietary tools

3. Consider Collaboration​

  • Choose tools that enable collaboration when needed
  • Balance individual focus vs. team needs
  • Use standard tools for easier onboarding

4. Focus on Creation​

  • Don't let tools become procrastination
  • Spend more time creating than organizing
  • Tools should enable, not constrain

5. Budget Wisely​

  • Many excellent free tools exist
  • Invest in tools that matter most
  • Consider one-time purchases over subscriptions

Minimal Viable Toolset​

If you want to start immediately with minimal tools:

  1. VS Code - Writing and coding
  2. Git + GitHub - Version control
  3. Blender - 3D modeling (free)
  4. Figma - Design (free tier)
  5. Notion - Organization (free tier)

This covers documentation, design, 3D modeling, and organization without cost.

Integration & Automation​

Useful Integrations​

  • Git hooks - Automate checks and formatting
  • GitHub Actions - Automated workflows
  • Zapier/Make - Connect different tools
  • API integrations - Custom connections

Automation Ideas​

  • Auto-deploy documentation on commit
  • Generate thumbnails from images
  • Sync tasks between tools
  • Backup automation
  • Social media scheduling

Getting Started Checklist​

For Stage 1, set up these essentials:

  • Choose and set up a markdown editor
  • Initialize Git repository
  • Create basic folder structure
  • Set up task tracking system
  • Choose sketching/design tool
  • Set up backup system
  • (Optional) Set up Notion or Obsidian for knowledge management
