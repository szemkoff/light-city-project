## üîß TECHNICAL SPECIFICATIONS

### **Image Formats by Use Case**

#### **Hero Images** (Large, Background)
- Format: WebP (primary), JPG (fallback)
- Resolution: 2560x1440px (2K) or 3840x2160px (4K)
- Compression: 80-85% quality
- File Size Target: <500KB (WebP), <800KB (JPG)

#### **Section Images** (Medium, Content)
- Format: WebP (primary), JPG (fallback)
- Resolution: 1600x900px (16:9) or 1600x1200px (4:3)
- Compression: 85-90% quality
- File Size Target: <300KB (WebP), <500KB (JPG)

#### **Card Images** (Small, Thumbnails)
- Format: WebP (primary), JPG (fallback)
- Resolution: 800x600px or 1200x800px
- Compression: 85-90% quality
- File Size Target: <150KB (WebP), <250KB (JPG)

#### **Diagrams/Illustrations** (Technical)
- Format: PNG (if transparency needed) or WebP
- Resolution: 1200x1200px (square) or 1600x900px (wide)
- Compression: Lossless (PNG) or 90% (WebP)
- File Size Target: <200KB

## üé® STYLE GUIDE FOR CONSISTENCY

### **Color Palette** (For AI Generation)
Always include these colors in prompts for brand consistency:

**Primary Colors**:
- Purple: `#667eea` to `#764ba2` (gradient)
- Gold: `#FFD700` (accents, highlights)
- Deep Black: `#0a0a0a` (backgrounds)
- Dark Blue-Black: `#1a1a2e` (surfaces)

**Secondary Colors**:
- Sky Blue: `#A0D8FF` (trust, clarity)
- Emerald Green: `#90EE90` (nature, growth)
- Coral Pink: `#F093FB` (warmth, community)

**Neutral Colors**:
- Light Gray: `#e0e0e0` (text)
- Medium Gray: `#b0b0b0` (secondary text)
- Dark Gray: `#808080` (muted elements)

### **Lighting Preferences**
- **Golden Hour**: Warm, soft, inspirational (most hero images)
- **Natural Daylight**: Clear, honest, professional (technical images)
- **Soft Interior**: Warm, inviting, cozy (interior spaces)
- **Dramatic Sky**: Volumetric rays, epic scale (visionary concepts)

### **Architectural Elements** (Always Include)
- Geodesic domes (triangular panels, 30-90m diameter)
- Cross-laminated timber (CLT) structure (visible wood grain)
- Sacred geometry patterns (Flower of Life, golden ratio, Metatron's Cube)
- Abundant plants (biophilic design, hanging gardens)
- Natural materials (wood, stone, glass, crystal)
- Solar integration (panels on roofs, BIPV)

### **Human Elements** (When Applicable)
- Diverse ages, ethnicities, backgrounds
- Comfortable, flowing clothing (modern bohemian, not costumes)
- Engaged in meaningful activities (meditation, gardening, building, learning)
- Natural expressions (joy, focus, connection, not posed)
- Small groups (3-5) or community gatherings (20-30)

### **Avoid** ‚ùå
- Religious symbols (crosses, specific deities)
- Overly futuristic/sci-fi (flying cars, holograms, aliens)
- Dystopian elements (dark, oppressive, industrial)
- Generic stock photo poses (fake smiles, awkward staging)
- Cluttered compositions (keep clean, spacious)
- Harsh lighting (prefer soft, natural)

## üîÑ WORKFLOW: FROM PROMPT TO WEBSITE

### **Step 1: Generate Image** (FluxFast 1.1)
```bash
# Using FluxFast 1.1 API (example)
curl -X POST https://api.fluxfast.ai/v1/generate \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "prompt": "YOUR_DETAILED_PROMPT_HERE",
    "model": "flux-fast-1.1",
    "width": 1600,
    "height": 900,
    "steps": 20,
    "guidance_scale": 7.5,
    "num_images": 4
  }'
```

### **Step 2: Select Best Image**
- Review 4 generated variations
- Choose best composition, lighting, accuracy
- Check for artifacts or errors

### **Step 3: Optimize for Web**
```bash
# Convert to WebP (using cwebp)
cwebp -q 85 original.jpg -o optimized.webp

# Create responsive sizes
convert original.jpg -resize 400x optimized-400w.jpg
convert original.jpg -resize 800x optimized-800w.jpg
convert original.jpg -resize 1200x optimized-1200w.jpg
convert original.jpg -resize 1600x optimized-1600w.jpg

# Convert all to WebP
for file in optimized-*.jpg; do
  cwebp -q 85 "$file" -o "${file%.jpg}.webp"
done
```

### **Step 4: Add to Project**
```bash
# Move to appropriate directory
mv optimized-*.webp /Users/stantheman/Light\ City\ Project/website/static/img/homepage/
mv optimized-*.jpg /Users/stantheman/Light\ City\ Project/website/static/img/homepage/

# Update git
cd /Users/stantheman/Light\ City\ Project
git add website/static/img/homepage/
git commit -m "Add generated image: [description]"
```

### **Step 5: Integrate into Homepage**
```tsx
// In index.tsx
<picture>
  <source
    srcset={`
      ${require('@site/static/img/homepage/optimized-400w.webp').default} 400w,
      ${require('@site/static/img/homepage/optimized-800w.webp').default} 800w,
      ${require('@site/static/img/homepage/optimized-1200w.webp').default} 1200w,
      ${require('@site/static/img/homepage/optimized-1600w.webp').default} 1600w
    `}
    type="image/webp"
  />
  <img
    src={require('@site/static/img/homepage/optimized-800w.jpg').default}
    alt="Descriptive alt text for accessibility and SEO"
    loading="lazy"
  />
</picture>
```

## üìù NOTES & BEST PRACTICES

### **Iteration Strategy**
1. **Generate 4 variations** per prompt
2. **Select best** based on composition, accuracy, mood
3. **Refine prompt** if needed and regenerate
4. **Optimize** for web (WebP, multiple sizes)
5. **Test** on actual homepage before committing

### **Quality Control Checklist**
- ‚úÖ Matches Light City aesthetic (purple/gold, sacred geometry)
- ‚úÖ Photorealistic or intentionally artistic (no uncanny valley)
- ‚úÖ No visible artifacts or errors
- ‚úÖ Proper lighting and composition
- ‚úÖ Diverse, inclusive representation (when humans present)
- ‚úÖ Technically accurate (architecture, technology)
- ‚úÖ Optimized file size (<500KB for large images)
- ‚úÖ Accessible (clear subject, good contrast)

### **Common Issues & Solutions**

**Issue**: Generated image too sci-fi/unrealistic  
**Solution**: Add "photorealistic," "architectural photography," "documentary style" to prompt

**Issue**: Wrong colors or mood  
**Solution**: Explicitly specify Light City colors (purple #667eea, gold #FFD700) and mood

**Issue**: Humans look awkward or fake  
**Solution**: Reference "candid photography," "documentary style," "natural expressions"

**Issue**: Architecture doesn't match specs  
**Solution**: Include specific dimensions (30-90m domes, CLT structure, triangular panels)

**Issue**: File size too large  
**Solution**: Compress to WebP at 80-85% quality, generate responsive sizes
